# Design System (기초 디자인 시스템)

> 비기술자도 이해할 수 있는 디자인 언어로 작성합니다.
> 최소 접근성 패키지를 포함합니다.

---

## MVP 캡슐

| # | 항목 | 내용 |
|---|------|------|
| 1 | 목표 | 수학 시험지를 AI로 분석하여 일관되고 객관적인 분석 결과 제공 |
| 2 | 페르소나 | 학원 운영자, 선생님 (교육업계 종사자) |
| 3 | 핵심 기능 | FEAT-1: 문항별 분석 (난이도, 유형, 정답률) |
| 4 | 성공 지표 (노스스타) | AI 분석 정확도/일관성 |
| 5 | 입력 지표 | 주간 분석 횟수, 사용자 만족도 |
| 6 | 비기능 요구 | 분석 결과 일관성 (같은 시험지 = 같은 맥락의 결과) |
| 7 | Out-of-scope | 외부 연동 (소셜로그인, 결제), 실시간 협업 |
| 8 | Top 리스크 | OCR 인식 정확도 (이미지/PDF에서 문제 추출 실패) |
| 9 | 완화/실험 | 수동 입력 폴백, 사용자 피드백으로 모델 개선 |
| 10 | 다음 단계 | 문항별 분석 MVP 개발 |

## 1. 디자인 철학

### 1.1 핵심 가치

| 가치 | 설명 | 구현 방법 |
|------|------|----------|
| **친근함** | 교육 앱답게 따뜻하고 접근하기 쉬운 느낌 | 부드러운 컬러, 둥근 모서리, 친절한 문구 |
| **신뢰감** | 분석 결과를 믿을 수 있다는 느낌 | 깔끔한 레이아웃, 명확한 데이터 표시, 일관된 UI |
| **단순함** | 복잡하지 않고 바로 사용할 수 있음 | 최소한의 입력, 직관적인 흐름, 군더더기 없는 디자인 |

### 1.2 참고 서비스 (무드보드)

| 서비스 | 참고할 점 | 참고하지 않을 점 |
|--------|----------|-----------------|
| 클래스팅 | 교육 앱의 친근한 느낌, 카드 기반 UI | 과한 애니메이션 |
| 노션 | 깔끔한 레이아웃, 명확한 정보 계층 | 너무 담백한 컬러 |
| 듀오링고 | 친근하고 응원하는 피드백 | 과한 게이미피케이션 |

### 1.3 디자인 원칙

1. **데이터 우선**: 분석 결과가 가장 잘 보여야 함
2. **학부모도 이해**: 전문 용어보다 쉬운 표현
3. **모바일 친화**: 모바일에서도 편하게 사용
4. **일관성**: 같은 요소는 같은 스타일

## 7. 접근성 체크리스트

### 7.1 필수 (MVP)

- [ ] **색상 대비**: 텍스트와 배경의 대비율 4.5:1 이상
  - Text Primary (#111827) on Background (#F9FAFB): 15.8:1 ✓
  - Text Secondary (#6B7280) on White: 5.0:1 ✓
- [ ] **포커스 링**: 키보드 탐색 시 포커스 표시 명확 (2px solid blue)
- [ ] **클릭 영역**: 버튼/링크 최소 44x44px
- [ ] **에러 표시**: 색상만으로 구분하지 않음 (아이콘/텍스트 병행)
- [ ] **폰트 크기**: 본문 최소 14px

### 7.2 권장 (v2)

- [ ] 키보드 전체 탐색 가능
- [ ] 스크린 리더 호환 (ARIA 라벨)
- [ ] 애니메이션 줄이기 옵션 (prefers-reduced-motion)
- [ ] 고대비 모드

## 8. 아이콘 & 일러스트

| 옵션 | 설명 | 선택 |
|------|------|------|

### 8.2 주요 아이콘

| 용도 | 아이콘 | Lucide 이름 |
|------|--------|-------------|

| 사용자 | 👤 | `User` |

- 버튼 내: 텍스트 왼쪽에 배치, 8px 간격

## 9. 마이크로 인터랙션

### 9.1 로딩 상태

```css
/* 분석 중 로딩 */
.analyzing-spinner {
  animation: spin 1s linear infinite;
}

/* 스켈레톤 로딩 */
.skeleton {
  background: linear-gradient(90deg, #E5E7EB 25%, #F3F4F6 50%, #E5E7EB 75%);
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
}
```

### 9.2 피드백 애니메이션

| 상황 | 애니메이션 |
|------|-----------|
| 파일 업로드 성공 | 체크 아이콘 + 초록 플래시 |
| 분석 완료 | 카드 슬라이드 인 |
| 에러 발생 | 빨간 쉐이크 |
| 버튼 클릭 | 살짝 눌림 (scale 0.98) |

### 9.3 트랜지션

- 기본 duration: 200ms
- easing: ease-out
- 호버/포커스 상태 변화에 적용

## 10. 톤 & 메시지

### 10.1 UI 문구 가이드

| 상황 | 예시 문구 | 톤 |
|------|----------|-----|
| 업로드 안내 | "시험지를 올려주세요" | 친절, 간결 |
| 분석 중 | "AI가 열심히 분석하고 있어요..." | 친근, 기다림 완화 |
| 분석 완료 | "분석이 완료되었습니다!" | 긍정적 |
| 에러 | "앗, 문제가 생겼어요. 다시 시도해주세요." | 사과, 해결책 제시 |
| 빈 상태 | "아직 분석한 시험지가 없어요. 첫 분석을 시작해보세요!" | 행동 유도 |

### 10.2 피해야 할 표현

- "Error 500" 같은 기술 용어
- "실패했습니다" (→ "문제가 생겼어요"로 대체)
- 명령조 ("하세요" → "해주세요", "해보세요")
