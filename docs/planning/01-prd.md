## 1. 문제 정의

### 1.1 현재 상황 (As-Is)

시험지 분석은 마치 **수작업으로 책 전체를 베끼는 것**과 같습니다:
- 선생님마다 분석 기준이 달라서 **일관성이 없음**
- 주관적인 분석으로 **신뢰도가 떨어짐**
- 같은 시험지를 다른 선생님이 분석하면 **전혀 다른 결과**가 나옴
- 분석 결과를 학부모에게 설명하기 어려움 - **언변이 좋은 선생님이 드묾**
- 시험 때마다 분석에 **많은 시간**이 소요됨

### 1.2 목표 상황 (To-Be)

AI가 **일관된 기준으로 분석**해주는 믿을 수 있는 조수가 됩니다:
- 같은 시험지는 **항상 같은 맥락의 분석 결과**
- 선생님 개인 분석과 AI 분석을 **비교하여 객관화**
- 학부모도 이해할 수 있는 **친근하고 명확한 리포트**
- 분석 시간 **대폭 절약** → 수업 준비에 집중

### 1.3 왜 지금인가? (Why Now)

- 교육 사업을 운영하면서 **실제로 분석 업무 부담**을 느끼고 있음
- AI 기술 발전으로 **이미지/텍스트 분석이 가능한 수준**에 도달
- 일관된 분석이 가능하다면 **교육 품질 향상**에 직접 기여

## 2. 사용자 페르소나

### 2.1 주요 페르소나

| 항목 | 내용 |
|------|------|
| 이름/별칭 | "바쁜 수학쌤" |
| 나이/직업 | 30대 학원 강사 / 원장 |
| 핵심 니즈 | 시험 분석 시간을 줄이고, 객관적인 피드백을 학부모에게 제공하고 싶음 |
| 불편 포인트 | 분석이 주관적이라 설득력이 떨어짐, 시간도 오래 걸림 |
| 사용 상황 | 시험 직후, 학부모 상담 전, 월간 학습 점검 시 |

### 2.2 보조 페르소나

| 항목 | 내용 |
|------|------|
| 이름/별칭 | "걱정되는 학부모" |
| 나이/직업 | 40대 학부모 |
| 핵심 니즈 | 아이의 수학 취약점을 정확히 알고 싶음 |
| 불편 포인트 | 선생님 설명이 추상적이거나 이해하기 어려움 |
| 사용 상황 | 상담 후 리포트 확인, 가정에서 학습 지도 시 |

### 2.3 페르소나의 하루 (시나리오)

> **오전 10시** - 김쌤은 어제 본 중2 수학 시험지 30장을 받습니다.
>
> **오전 10시 5분** - 시험지 사진을 찍어 "수학 시험지 분석기"에 업로드합니다.
>
> **오전 10시 10분** - AI가 각 문항의 난이도, 유형, 학생별 정답률을 분석합니다.
>
> **오전 10시 15분** - "민수는 도형 문제에서 약하고, 서연이는 계산 실수가 잦네요" 라는 분석 리포트를 확인합니다.
>
> **오후 2시** - 학부모 상담에서 AI 분석 리포트를 보여주며 "민수는 도형 단원 복습이 필요합니다"라고 구체적으로 설명합니다.
>
> **저녁** - 김쌤은 분석에 쓰던 2시간을 아꼈고, 다음 수업 준비에 집중합니다.

## 3. 사용자 스토리

### FEAT-0: 공통 흐름 (온보딩/로그인)
- [ ] 사용자로서, 간편하게 가입할 수 있어야 한다. 왜냐하면 **복잡한 가입 절차는 사용을 포기하게 만들기 때문**.
- [ ] 사용자로서, 로그인 상태가 유지되어야 한다. 왜냐하면 **매번 로그인하면 번거롭기 때문**.

### FEAT-1: 문항별 분석 (MVP)
- [ ] 선생님으로서, 시험지 이미지/PDF를 업로드하면 **각 문항의 난이도, 유형, 배점을 자동 분석**받을 수 있어야 한다. 왜냐하면 **수동 분석 시간을 줄이기 위해**.
- [ ] 선생님으로서, 학생 답안지를 업로드하면 **문항별 정답/오답과 정답률을 확인**할 수 있어야 한다. 왜냐하면 **어떤 문제가 어려웠는지 파악하기 위해**.
- [ ] 선생님으로서, **같은 시험지를 다시 분석해도 일관된 결과**를 받을 수 있어야 한다. 왜냐하면 **분석 신뢰도를 확보하기 위해**.

### FEAT-2: 시험지 OCR 인식 (v2)
- [ ] 사용자로서, 손글씨나 인쇄된 시험지 이미지에서 **문제와 답안이 자동 추출**되어야 한다.

### FEAT-3: 학생 취약점 리포트 (v2)
- [ ] 선생님으로서, 학생별 **틀린 문제 패턴을 분석한 취약점 리포트**를 받을 수 있어야 한다.
- [ ] 학부모로서, 아이의 **취약 단원과 추천 학습 방향**을 이해하기 쉬운 언어로 받을 수 있어야 한다.

### FEAT-4: 학급/클래스 전체 분석 (v2)
- [ ] 원장으로서, **클래스 전체의 평균, 분포, 취약 문항**을 한눈에 볼 수 있어야 한다.

## 4. 성공 지표

### 4.1 노스스타 지표 (결과 지표)

| 지표명 | 정의 | 목표치 | 측정 주기 |
|--------|------|--------|----------|
| AI 분석 정확도 | 사용자가 "분석이 맞다"고 평가한 비율 | 80% 이상 | 월간 |
| 분석 일관성 | 같은 시험지 재분석 시 핵심 결과 일치율 | 95% 이상 | 주간 |

### 4.2 입력 지표 (행동 지표)

| 지표명 | 정의 | 목표치 |
|--------|------|--------|
| 주간 분석 횟수 | 사용자당 주간 시험지 분석 횟수 | 3회 이상 |
| 사용자 만족도 | 분석 결과에 대한 별점 평가 | 4.0/5.0 이상 |

## 5. 범위 관리

### 5.1 MVP (반드시 포함)
- **FEAT-1**: 문항별 분석 (난이도, 유형, 정답률)
- **FEAT-0**: 기본 회원가입/로그인
- 이미지/PDF 업로드 기능
- 분석 결과 조회 화면

### 5.2 옵션 (시간 허용 시)
- 빈 시험지 난이도 분석 (학교 시험 난이도 파악용)
- 분석 결과 PDF 내보내기

### 5.3 Non-goals (명시적 제외)
- **외부 연동**: 소셜 로그인, 결제 시스템 - MVP 이후 검토
- **실시간 협업**: 여러 선생님이 동시 편집 - 필요성 검증 후
- **모바일 네이티브 앱**: 웹 반응형으로 우선 대응
- **자동 채점**: 문항 분석에 집중, 채점은 기존 방식 유지

## 6. 가정과 리스크

### 6.1 핵심 가정

| # | 가정 | 검증 방법 |
|---|------|----------|
| 1 | 시험지 이미지에서 문항을 추출할 수 있다 | OCR + AI 테스트 |
| 2 | AI가 수학 문항의 유형/난이도를 판단할 수 있다 | LLM 프롬프트 테스트 |
| 3 | 같은 입력에 일관된 결과를 낼 수 있다 | Temperature 조정, 캐싱 테스트 |
| 4 | 선생님들이 기존 방식보다 편리하다고 느낀다 | 사용자 인터뷰 |

### 6.2 Top 5 리스크 보드

| # | 가정 | 리스크 | 완화 | 트리거 | 대응 |
|---|------|--------|------|--------|------|
| 1 | OCR 정확도 | 이미지에서 문제 추출 실패 | 수동 입력 폴백 UI | 추출 실패율 > 30% | 이미지 품질 가이드 제공 |
| 2 | 분석 일관성 | 같은 시험지인데 다른 결과 | 분석 결과 캐싱, 시드 고정 | 재분석 불일치 > 5% | 분석 버전 관리 |
| 3 | 분석 정확도 | AI가 문항 유형을 잘못 판단 | 사용자 피드백으로 개선 | 정확도 < 70% | 도메인 전문가 검수 |
| 4 | 사용자 채택 | 기존 방식을 바꾸려 하지 않음 | 무료 체험, 시간 절약 강조 | MAU < 목표 50% | 사용성 개선 |
| 5 | 응답 속도 | 분석이 너무 오래 걸림 | 비동기 처리, 진행률 표시 | 분석 > 30초 | 모델 최적화 |

## 7. 해결 가설과 실험

### 7.1 가설

> 만약 **시험지를 업로드하면 AI가 일관된 기준으로 문항을 분석**해준다면,
> **선생님의 분석 시간이 50% 이상 줄어들고, 학부모 피드백 만족도가 높아질 것**이다.
> 왜냐하면 **객관적이고 일관된 분석 결과는 설득력이 있기 때문**이다.

### 7.2 실험 루프

| 단계 | 내용 |
|------|------|
| 실험 | MVP로 10명의 선생님에게 실제 시험지 분석 경험 제공 |
| 관측 | 분석 소요 시간, 결과 만족도, 일관성 평가 |
| 학습 | 분석 시간 50% 단축, 만족도 4.0 이상이면 성공 |
| 다음 | 성공 시 FEAT-3(취약점 리포트) 추가, 실패 시 분석 품질 개선 |
